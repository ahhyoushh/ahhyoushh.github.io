C++ Anti-Patterns
Every mistake here is a real bug in a production codebase somewhere.
WrongRightWhy
vector<T> by value in paramsconst vector<T>&Copies entire vector on every call. Hidden O(n) cost.
(lo + hi) / 2lo + (hi - lo) / 2Overflows INT_MAX. Undefined behavior.
int with n > 10⁴ multiplicationlong long10⁵ × 10⁵ = 10¹⁰ silently overflows 32-bit int.
cout << endlcout << "\n"endl flushes the buffer. "\n" doesn't. Slow in loops.
i < v.size() with int ii < (int)v.size()size() is unsigned. Comparing signed/unsigned causes subtle underflow bugs.
unordered_set for everythingsort + two pointers when space mattersunordered_set has ~150MB overhead on large inputs.
stack<int> default backendstack<int, vector<int>>Default deque backend is heap-fragmented. vector = contiguous = cache friendly.
unordered_set without reserveseen.reserve(nums.size())Multiple rehashes without reserve. Reserve eliminates them entirely.

Phase 3 — Edge AI

ONNX Runtime C++ API
onnxruntime.ai/docs/api/c
Microsoft's cross-platform inference runtime. Load ONNX models, run in C++. Has an ARM build for Pi deployment.
Phase 3 · Primary Runtime
TensorRT Developer Guide
docs.nvidia.com/tensorrt
FP16/INT8 quantization, operator fusion, engine serialization. Read Chapters 1–5 before touching code.
Phase 3 · GPU Optimization
pybind11
pybind11.readthedocs.io
C++↔Python bridge. Write inference in C++, call from Python with numpy. Buffer protocol section = zero-copy transfer.
Phase 3 · Python Bridge
CUDA C++ Guide
docs.nvidia.com/cuda
Thread hierarchy, shared vs global memory, warp execution. Start at Chapter 1, implement as you read.
Phase 3 · GPU Kernels
GStreamer
gstreamer.freedesktop.org
Camera feed pipeline on Linux edge devices. Harder than the AI itself. Essential for production edge deployments.

Docker Docs
docs.docker.com
Every edge AI deployment uses containers. Docker + Docker Compose for multi-service pipelines. ARM builds supported.
